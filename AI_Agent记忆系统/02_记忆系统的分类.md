## 内容讲解

### 从人脑到 Agent：三种记忆

上一课我们提到，AI Agent 的记忆系统设计大量借鉴了认知科学。现在我们正式展开这个框架。

| 记忆类型 | 人脑类比 | Agent 中的对应 | 容量 | 持续时间 |
|----------|----------|---------------|------|----------|
| **短期记忆** | 你正在听的一句话 | 当前对话的上下文窗口 | 有限（如 128K tokens） | 单次会话 |
| **长期记忆** | 你记得的知识和经历 | 外部持久化存储（向量数据库、文件等） | 近乎无限 | 跨会话持久化 |
| **工作记忆** | 你心算时脑中同时操作的几个数字 | Agent 的中间推理状态（Scratchpad） | 很小 | 当前任务期间 |

### 一、短期记忆（Short-term Memory）

**人脑类比**：别人刚告诉你一个电话号码，你还没存到通讯录里。

**在 Agent 中的实现**：就是当前对话的**上下文窗口**（Context Window）。

```
┌────────────────────────────────────┐
│         上下文窗口 (128K)          │
│                                    │
│  系统提示词                         │
│  用户消息 1                         │
│  Agent 回复 1                      │
│  用户消息 2                         │
│  Agent 回复 2（含工具调用）          │
│  工具返回结果                       │
│  Agent 回复 3                      │
│  ...                               │
│  （越来越满...）                    │
│                                    │
└────────────────────────────────────┘
```

**特点**：
- 信息"进来"很自然（每轮对话自动追加）
- 容量有硬上限
- **会话结束即消失**
- 上下文越满，模型性能越差（你在第 1 课已经指出了这一点）

**常见管理策略**：
- **滑动窗口**：只保留最近 N 轮对话，丢弃更早的
- **摘要压缩**：把旧对话压缩成摘要，腾出空间
- **选择性保留**：只保留"重要"的消息（如用户指令），丢弃"不重要"的（如中间工具输出）

### 二、长期记忆（Long-term Memory）

**人脑类比**：你知道自己家的地址、会骑自行车、记得去年生日怎么过的。

长期记忆又可以细分为两种（同样借鉴认知科学）：

#### 陈述性记忆（Declarative / Explicit）

可以用语言描述的知识：

- **语义记忆**（Semantic）：一般性知识和事实
  - 例：*"用户是一名 Python 开发者，使用 macOS"*
  - 实现：知识库、用户画像数据库

- **情景记忆**（Episodic）：特定事件和经历
  - 例：*"2026年2月20日，用户让我帮忙调试了一个内存泄漏问题，最后发现是闭包导致的"*
  - 实现：对话历史存储、事件日志

#### 程序性记忆（Procedural / Implicit）

"怎么做"的知识，通常难以用语言描述：

- 例：*"处理用户的代码审查请求时，先看整体架构再看细节"*
- 实现：Agent 的工具调用策略、微调过的行为模式

在当前的 Agent 系统中，**陈述性记忆**有比较成熟的实现方案（向量数据库 + RAG），而**程序性记忆**仍然主要靠提示词工程和模型微调，是一个活跃的研究方向。

### 三、工作记忆（Working Memory）

**人脑类比**：你在心算 `47 × 23` 时，需要同时在脑中保持 `47 × 3 = 141`、`47 × 20 = 940`，然后把它们加起来。这些中间结果就在你的"工作记忆"中。

工作记忆 ≠ 短期记忆。区别在于：

| | 短期记忆 | 工作记忆 |
|---|---|---|
| 功能 | **存储**信息 | **存储 + 操作**信息 |
| 类比 | 一张便利贴 | 一块白板（可以在上面写写画画） |
| Agent 中 | 上下文窗口中的被动信息 | Agent 主动使用的推理空间 |

**在 Agent 中的实现**：

- **Scratchpad / 草稿板**：Agent 把中间推理结果写在一个专门的区域
- **思维链（Chain of Thought）**：模型生成中间推理步骤
- **任务状态追踪**：记录"当前在第几步，前面的结果是什么"

一个研究 Agent 的工作记忆可能像这样：

```
[Scratchpad]
任务：对比论文 A 和论文 B 的方法论差异
当前进度：步骤 3/5

已完成：
- 论文 A 使用监督学习，数据集 10K 样本
- 论文 B 使用自监督学习，数据集 500K 样本

待完成：
- 对比训练策略
- 对比评估指标
- 撰写总结
```

### 三种记忆的协作

它们不是孤立的，而是协同工作：

```
用户输入："帮我写一封邮件给张总，关于上次讨论的项目进展"
    │
    ▼
[短期记忆] 当前对话里有没有提到"张总"或"项目"？
    │ 没有
    ▼
[长期记忆] 搜索历史：找到了！
    │ - 语义记忆：张总是客户方技术负责人
    │ - 情景记忆：上周三讨论了数据迁移项目，决定用方案 B
    ▼
[工作记忆] 整合信息，规划邮件结构：
    │ - 开头：问候 + 引用上次会议
    │ - 正文：方案 B 的进展更新
    │ - 结尾：下一步计划
    ▼
生成邮件输出
```

## 小结

1. Agent 记忆系统分为**短期记忆**（上下文窗口）、**长期记忆**（持久化存储）和**工作记忆**（中间推理空间）三种。
2. **短期记忆**容量有限、会话级别，需要通过滑动窗口、摘要压缩等策略管理。
3. **长期记忆**进一步分为陈述性（语义 + 情景）和程序性，目前陈述性记忆的实现较成熟，程序性记忆仍是研究热点。
4. **工作记忆 ≠ 短期记忆**——工作记忆不仅存储，还包含主动操作和推理。
5. 三种记忆**协同工作**：短期记忆捕获当前输入，长期记忆提供背景知识，工作记忆整合信息并推理。

## 我的答案与纠偏记录

**掌握判定：基本掌握**

### 问题 1 我的回答
a)短期记忆 b)工作记忆 c)长期记忆

**纠偏**：(b)(c) 正确。**(a) 需要修正**——"月薪2万"虽然初始进入短期记忆（当前对话上下文），但它的本质是一个**持久化事实**，理财助手下个月、明年都需要用到它，应当被固化为**长期记忆**（语义记忆——用户画像中的收入信息）。关键区分：信息"当前在哪"和"应该持久化到哪"是两回事。

### 问题 2 我的回答
摘要压缩策略的意思是,会话上下文窗口有限,tokens越多模型幻觉越厉害,为了减少幻觉,以及在会话超过上限的时候,会话能够继续运行,需要根据不同的策略压缩之前的对话内容,从而减少tokens。但是这个策略引入了新的问题是,AI会"忘记"或者"忽略"对话中的一些细节。

**纠偏**：完全正确。补充：摘要压缩还会引入**摘要本身的偏差**——模型在压缩时可能改变信息的重点或遗漏某些上下文关联，导致后续推理基于一个"走样的"历史。

### 问题 3 我的回答
如果没有工作记忆,执行复杂任务时,就需要将大量的中间过程说出来,写到短期记忆中,这样会导致tokens迅速膨胀,在解决复杂任务的时候,容易出现忘记某些细节和步骤的问题。

**纠偏**：基本正确。更核心的原因是：工作记忆提供的是**结构化的任务状态管理**，而不仅仅是"多一个地方存东西"。没有工作记忆的 Agent 就像一个人不能在纸上列提纲就直接写长文——容易逻辑混乱、遗漏分支、重复劳动。
